2. 웹의 기초

1) 웹 리소스의 구성
HTML (마크업 언어) + CSS (스타일 언어) + JS(기능 구현)

2) http(80) 프로토콜 : 클라이언트와 웹서버 사이의 통신 프로토콜

Request와 Response로 구성

Request
Method + Path + Version
Header
Body (없을 수도 있음)

메소드의 종류
Get, Post, Head, Trace, Delete, Put, Patch, Options

Response
Version + Return Code
Header
Body

Return Code
200 : 성공
300 : 위치변경
400 : 반환불가
500 : 서버불능

3) 쿠키와 세션
쿠키 : 웹브라우저 측에 저장되는 인증정보 (Key = Value)
세션 : 서버 측에 저장되는 인증정보 (랜덤문자열)

4) 도메인
외우기 어려운 IP를 의미를 가진 문자열에 대응

네임서버 질의(nslookup) => 주소 반환 => 반환된 주소에 접속
(상위 네임서버에서 하위 네임서버로 계단식)
DNS 캐시를 이용해 서버 부하 경감


5) 웹 서비스 구성

클라이언트 <=Request, Response=> 웹 서버 <=동적 페이지=> 웹 어플리케이션 <=데이터 조작=> DBMS

7) 기타
Https(443) : 보안 웹 프로토콜
URI : 실제 파일 위치를 나타내는 URL을 포함하는 개념으로 추상적인 위치까지 지정
Encoding과 Encryption의 차이 : 전자는 키가 없이 모두가 해독 가능, 후자는 유효한 키를 가져야 해독 가능
Encoding은 예약어나 기호들이 문법적인 용도 외로 사용할 때 쓰임

3. 웹 해킹 개요

1) 해킹 : 본래의 의도와는 다른 동작을 일으키거나 주어진 권한 이상으로 정보를 열람, 복제, 변경하는 행위

2) 웹 해킹의 분류
Client-Side Attack - 서비스 사용자를 공격하여 다른 사용자의 웹 브라우저에 원하는 행위
Server-Side Attack - 웹 서버를 공격하여 정보를 유출하거나 서버를 탈취하는 행위
